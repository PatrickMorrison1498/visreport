About Vis.Report

I would like the acknowledge the Whadjuk Noongar people, whose boodja (country) this site was built on. I would also like acknowledge the Noongar peoples whose seas this site exists for: the Whadjuk, Amangu, Yuat, Pinjarup, Wardandi, Bibbulman and Minang. The Noongar peoples have ongoing connections to the sea: a connection which began at a time when sea levels were over 80 metres lower than present, and all the sites listed here were dry land. I pay my respect to Elders past, present and emerging, and encourage all divers to reflect on the signficance of this beautiful sea country.

Vis.Report contains a database of visibility and condition reports for dive sites around Perth, all contributed by users. Clicking any site on the map will display the last three reports. Clicking on those will show a list of all reports for that site, and allow you to contribute your own (if you are signed in). Please share it around, the more people that contribute frequently, the more useful the information becomes! Dive sites are curated by me. If you have a suggestion for a site that isn't on here, please email vis.report.app@gmail.com.

The factors affecting dive conditions and visibility are not yet fully understood, particularly in the Swan River. I intend to use this data to better understand long-term visibility trends. Any publication of the data will acknowledge that it was contributed by citizen scientists via this platform.

All data contributed is licensed under a Creative Commons Attribution-ShareAlike 4.0 International License. This means it is open for research or any other derivitive work, as long as it properly attributed and that work is also shared with the same conditions. JSON files for the sites and reports are dynamically generated straight from the database. These contain exactly the same information as is visible on the site, except username is replaced with a unique number.

This site is built on the Django web framework, connected to a PostgreSQL database with a PostGIS extension. All passwords are stored as a secure hash using Django's authentication system. All code can be found on the Github page.

Safe diving!

Patrick Morrison
